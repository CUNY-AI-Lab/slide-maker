Resource Panel Alternatives

Three alternative designs for the right-hand resources panel (240px). Each preserves the four resource types (Files, Templates, Artifacts, Themes) while exploring different organizational paradigms.

A — Accordion Stack
All four sections visible simultaneously as collapsible regions. No tab switching — expand what you need. Templates auto-expand as the primary resource.

Strengths

  • All sections always visible — no tab blindness
  • Section headers show item counts at a glance
  • Natural priority: expanded = important
  • Minimal cognitive shift between sections

Trade-offs

  • Collapsed sections are just headers — zero preview
  • Only one section can practically be expanded at full height
  • Vertical space split means less room per section than tabs

Strengths

  • All resource types in one searchable list — fast for power users
  • Pinned shelf gives one-click access to favorites
  • Filter chips let users narrow by type when needed
  • Keyboard-friendly — / to focus, arrows to navigate

Trade-offs

  • Mixed list can overwhelm if many resources exist
  • Less visual for templates (no thumbnail grid)
  • Requires typing — slower for browse-oriented users
  • Pin management adds a layer of configuration
C — Icon Dock + Context View
Vertical icon rail on the right edge works like a mini dock. Click an icon to open its flyout. When nothing is selected, shows contextual suggestions based on the active slide.

Strengths

  • Context view surfaces relevant resources for the active slide
  • Dock rail is always visible — one-click to any section
  • Badges on dock icons show file counts
  • Suggested view reduces decision fatigue for new users
  • Star icon as default = "smart home" for the panel

Trade-offs

  • Dock rail eats ~42px of the 240px width
  • Suggestion logic needs active slide awareness (new coupling)
  • More complex to implement than tabs or accordion
  • Users may expect full width panel when clicking dock item
