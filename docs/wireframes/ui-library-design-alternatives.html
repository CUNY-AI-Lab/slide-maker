Hybrid: Accordion + Cmd+K Bits UI

Best of both worlds. The accordion serves as the always-visible browsable panel (Option A), while Cmd+K opens the command palette for cross-type search (Option B). Users who know what they want hit Cmd+K; users who are exploring browse the accordion.

6. Toast Notifications UX feedback (implicit in all TODO areas)

Currently: no toast system. Actions like export, delete, share, and mutation errors use console.error or inline state flags. Svelte Sonner provides a drop-in toast system with auto-dismiss, stacking, and swipe-to-dismiss.

Where to use: Export success/failure. Slide deleted (with undo action). Theme applied. Share link copied. Mutation rejected. File upload complete. Token cap warning.

Setup: One <Toaster /> in the root layout. Call toast.success('...') anywhere. Custom styling via unstyled prop + CSS variables. ~3KB gzipped.

Migration Path

Incremental adoption — each library can be introduced independently. No big-bang rewrite.

Phase 1: Foundation
Install Bits UI and Svelte Sonner. Add <Toaster /> to root layout. Replace NewDeckDialog and ShareDeckDialog with Bits UI Dialog. Remove a11y_* suppressions from those files.
Phase 2: Resource Panel
Replace ResourcePanel tab bar with Bits UI Accordion. Migrate each tab component into an accordion section (minimal changes — same content, new wrapper). Add Cmd+K palette using Bits UI Dialog + Combobox.
Phase 3: Micro-interactions
Replace all title attributes with Bits UI Tooltip. Wire toast calls into export, delete, share, and mutation flows. Replace ModulePicker overlay with Bits UI Popover. Add Floating UI for module toolbar positioning.
Phase 4: Accessibility Audit
Remove remaining a11y_* suppressions. Verify keyboard navigation end-to-end (Tab, arrow keys, Escape, Enter). Run pnpm audit:a11y for contrast compliance. Add SplitHandle keyboard resizing using Bits UI Slider primitive.
